setPacketSize 메서드(SQLServerDataSource)

SQL Server와의 통신에 사용되는 현재 네트워크 패킷 크기(바이트)를 설정합니다.

구문

public void setPacketSize(int packetSize)

매개 변수

packetSize

네트워크 패킷 크기가 들어 있는 int 값입니다.

주의

이 속성에 허용되는 값 범위는 [-1 | 0 | 512..32767]입니다. 이 속성을 허용 범위 외의 값으로 설정하면 예외가 발생합니다.

응용 프로그램에서는 SSL(Secure Sockets Layer) 암호화를 사용하여 연결되어 있는 동안 packetSize 속성을 설정할 수 있습니다. SQL Server용 Microsoft JDBC Driver에서는 서버와 패킷 크기를 협상합니다. encrypt 속성이 "true"로 설정되어 있으며 협상된 패킷 크기가 JVM(Java Virtual Machine)의 기본 보안 공급자가 제공하는 SSL 레코드 크기보다 큰 경우에는 드라이버에서 오류가 발생하고 연결이 종료됩니다.

또한 응용 프로그램에서 SSL 암호화를 요청하지 않고 packetSize 속성을 설정할 수도 있습니다. 이 경우 서버의 요구 사항에 따라 클라이언트에서 SSL 암호화를 지원해야 하면 드라이버에서는 JVM의 기본 보안 공급자가 제공하는 SSL 레코드 크기를 확인합니다. packetSize 속성이 JVM의 기본 보안 공급자가 제공하는 SSL 레코드 크기보다 크면 드라이버에서 오류가 발생하고 연결이 종료됩니다.

SSL을 사용하는 방법에 대한 자세한 내용은 SSL 암호화 사용을 참조하십시오.
