JDBC 드라이버의 새로운 기능

SQL Server용 Microsoft JDBC Driver 4.0에는 새로운 기능과 향상된 기능이 여러 가지 포함되어 있습니다.

SQL Azure 데이터베이스 연결 정보

이제 SQL Azure 데이터베이스 연결 정보에 대한 항목이 있습니다. 자세한 내용은 SQL Azure의 데이터베이스에 연결을 참조하십시오.

고가용성 및 재해 복구 지원

SQL Server용 Microsoft JDBC Driver 4.0은 SQL Server 2012에서 AlwaysOn 가용성 그룹에 대한 고가용성 재해 복구 연결을 지원합니다. 자세한 내용은 고가용성, 재해 복구를 위한 JDBC 드라이브 지원을 참조하십시오.

Kerberos 통합 인증을 사용하여 SQL Server에 연결

SQL Server용 Microsoft JDBC Driver 4.0에는 SQL Server 데이터베이스에 연결하는 응용 프로그램을 위한 유형 4 Kerberos 통합 인증이 있습니다. 자세한 내용은 Kerberos 통합 인증을 사용하여 SQL Server에 연결을 참조하십시오. (유형 2 Kerberos 통합 인증은 SQL Server용 Microsoft JDBC Driver의 4.0 이전 버전에서 사용할 수 있습니다.)

확장 이벤트 로그의 진단 정보에 액세스

연결 실패 이유를 파악하기 위해 서버의 확장 이벤트 로그의 정보에 액세스할 수 있습니다. 자세한 내용은 확장 이벤트 로그의 진단 정보에 액세스를 참조하십시오.

스파스 열에 대한 추가 지원

응용 프로그램에서 스파스 열을 사용하는 테이블의 데이터에 이미 액세스하는 경우 성능이 향상된 것을 알 수 있어야 합니다. getColumns 메서드(SQLServerDatabaseMetaData)에서 열에 대한 정보(스파스 열 정보 포함)를 가져올 수 있습니다. SQL Server 스파스 열에 대한 자세한 내용은 스파스 열 사용을 참조하십시오.

결과 집합의 열이 스파스 열 집합인지 여부를 나타내기 위해 isSparseColumnSet 메서드가 SQLServerResultSetMetaData 클래스에 추가되었습니다.

Xid.getFormatId

SQL Server용 Microsoft JDBC Driver 4.0 이전에는 JDBC 드라이버가 응용 프로그램에서 형식 식별자를 받았지만 해당 값을 데이터베이스 서버에 전달하지 않았습니다. 대신 JDBC 드라이버는 형식 식별자 0을 데이터베이스 서버에 전달했습니다.

SQL Server용 Microsoft JDBC Driver 4.0부터는 JDBC 드라이버가 응용 프로그램의 형식 식별자를 데이터베이스 서버에 전달합니다. 업데이트된 동작을 가져오려면 서버에서 sqljdbc_xa.dll이 업데이트되었는지 확인하십시오. sqljdbc_xa.dll의 업데이트된 버전을 서버에 복사하는 방법에 대한 자세한 내용은 XA 트랜잭션 이해를 참조하십시오.

Itanium에서는 JDBC 드라이버 4.0 응용 프로그램을 지원하지 않음

SQL Server용 Microsoft JDBC Driver 4.0 응용 프로그램을 Itanium 컴퓨터에서 실행하는 것은 지원되지 않습니다.
